## 📁 プロジェクトの追加方法

### ステップ1: テンプレートをコピー

1. **index.html** をテキストエディタで開く
2. プロジェクト2のブロック全体を探す：
   ```html
   <!-- 📌 プロジェクト 2：データベース連携Webアプリ -->
   ```
   から
   ```html
   <!-- 📌 プロジェクト 2 ここまで -->
   ```
   まで
3. この部分を**すべて**コピー

### ステップ2: 貼り付け

1. コピーしたブロックを、プロジェクト2の**下**に貼り付け
2. 貼り付けた場所：
   ```html
   <!-- 📌 プロジェクト 2 ここまで -->
   
   ← ここに貼り付け！
   
   <!-- ✨ プロジェクト 3 を追加したい場合 -->
   ```

### ステップ3: 番号を変更

貼り付けたブロックの以下を変更：

```html
<!-- 変更前 -->
<!-- 📌 プロジェクト 2：データベース連携Webアプリ -->

<!-- 変更後 -->
<!-- 📌 プロジェクト 3：新しいプロジェクト名 -->
```

最後のコメントも変更：
```html
<!-- 変更前 -->
<!-- 📌 プロジェクト 2 ここまで -->

<!-- 変更後 -->
<!-- 📌 プロジェクト 3 ここまで -->
```

### ステップ4: 内容を編集

次の「プロジェクトの編集方法」に従って、内容を編集します。

## ✏️ プロジェクトの編集方法

### 1. タイトルの編集

**探す場所：**
```html
<!-- ✏️ プロジェクトタイトル（ここを編集） -->
<h3 class="text-3xl font-bold text-dark mb-4">データベース連携Webアプリ 🌐</h3>
```

**編集例：**
```html
<h3 class="text-3xl font-bold text-dark mb-4">SNS風Webアプリ 📱</h3>
```

**絵文字の選び方：**
- 🤖 IoT・組み込み系
- 🌐 Web アプリ
- 📱 モバイルアプリ
- 💻 デスクトップアプリ
- 🎨 デザイン系
- 📊 データ分析系
- 🎮 ゲーム系

### 2. 使用技術タグの編集

**探す場所：**
```html
<!-- ✏️ 使用技術タグ（ここを編集・追加・削除） -->
<div class="flex flex-wrap gap-2">
    <span class="tag-chip">Node.js</span>
    <span class="tag-chip">Express</span>
    <span class="tag-chip">MySQL</span>
</div>
```

**タグの追加方法：**
```html
<!-- 新しいタグを追加 -->
<span class="tag-chip">React</span>
<span class="tag-chip">TypeScript</span>
```

**タグの削除方法：**
```html
<!-- 削除したいタグを探して、その行を削除 -->
<span class="tag-chip">削除したい技術</span> ← この行を削除
```

### 3. Role（担当範囲）の編集

**探す場所：**
```html
<!-- ✏️ Role（担当範囲） - ここを編集 -->
<p class="text-text-main leading-relaxed">
    バックエンド開発、データベース設計、API設計
</p>
```

**編集例：**
```html
<p class="text-text-main leading-relaxed">
    フロントエンド開発、UIデザイン、ユーザーテスト
</p>
```

### 4. Background（背景・課題）の編集

**探す場所：**
```html
<!-- ✏️ Background（背景・課題） - ここを編集 -->
<p class="text-text-main leading-relaxed mb-2">
    <strong>目的：</strong>CRUD操作を実装した実践的なWebアプリケーション開発
</p>
<p class="text-text-main leading-relaxed mb-2">
    <strong>課題：</strong>データの永続化と効率的な管理が必要
</p>
<p class="text-text-main leading-relaxed">
    <strong>解決方法：</strong>ORMを使用してデータベース操作を簡潔に記述
</p>
```

**編集のコツ：**
- **目的**: なぜ作ったのか、何を目指したのか
- **課題**: どんな問題があったのか
- **解決方法**: どうやって解決したのか

**良い例：**
```html
<p class="text-text-main leading-relaxed mb-2">
    <strong>目的：</strong>学校のイベント情報を一元管理し、学生の参加率を向上させる
</p>
<p class="text-text-main leading-relaxed mb-2">
    <strong>課題：</strong>従来は紙のチラシで告知していたため、情報が届きにくかった
</p>
<p class="text-text-main leading-relaxed">
    <strong>解決方法：</strong>スマホから簡単にアクセスできるWebアプリを開発し、プッシュ通知機能を実装
</p>
```

### 5. Result（成果）の編集

**探す場所：**
```html
<!-- ✏️ Result（成果） - ここを編集（項目の追加・削除OK） -->
<ul class="text-text-main leading-relaxed space-y-2">
    <li>• Node.js + Expressでサーバー構築を実現</li>
    <li>• Prisma ORMでデータベース操作を効率化</li>
    <li>• CRUD操作の完全な実装に成功</li>
</ul>
```

**項目の追加方法：**
```html
<li>• 新しい成果を追加</li>
```

**項目の削除方法：**
```html
<!-- 削除したい項目の行を削除 -->
<li>• 削除したい項目</li> ← この行を削除
```

**書き方のコツ：**
- 具体的に書く
- 数字を入れる
- 「〜した」「〜を実現」と明確に

**良い例：**
```html
<li>• ページ読み込み速度を2秒→0.5秒に短縮</li>
<li>• ユーザー登録数が3ヶ月で200人突破</li>
<li>• モバイル対応により、スマホからのアクセスが60%増加</li>
```

### 6. Reflection（学び・改善点）の編集

**探す場所：**
```html
<!-- ✏️ Reflection（学び・改善点） - ここを編集 -->
<p class="text-text-main leading-relaxed mb-2">
    <strong>学んだこと：</strong>フルスタック開発の流れ、データベース設計の重要性
</p>
<p class="text-text-main leading-relaxed">
    <strong>改善点：</strong>エラーハンドリングの強化と、セキュリティ対策の実装
</p>
```

**書き方のコツ：**
- 技術的な学び
- チームワークでの学び
- 次にどうするか

**良い例：**
```html
<p class="text-text-main leading-relaxed mb-2">
    <strong>学んだこと：</strong>Reactのstate管理の重要性、ユーザーテストの価値、チーム開発でのGit運用
</p>
<p class="text-text-main leading-relaxed">
    <strong>改善点：</strong>次回はTypeScriptを導入してバグを減らしたい。また、テストコードを書く習慣をつけたい
</p>
```

### 7. リンクボタンの編集

**探す場所：**
```html
<!-- ✏️ 詳細ページへのリンク（href="#" を変更） -->
<a href="#" class="inline-flex items-center gap-2 px-6 py-3 bg-primary text-white font-medium rounded-full hover:bg-dark transition-colors mt-4">
    詳しく見る
```

**リンク先の変更方法：**
```html
<!-- 変更前 -->
<a href="#" ...>

<!-- 変更後（詳細ページへリンク） -->
<a href="project1.html" ...>
```

**ボタンのテキスト変更：**
```html
<!-- 変更前 -->
詳しく見る

<!-- 変更後 -->
デモを見る
もっと見る
詳細はこちら
```

**ボタンを削除する場合：**
```html
<!-- この <a>...</a> 全体を削除 -->
```

## 🗑️ プロジェクトの削除方法

### ステップ1: 削除範囲を確認

削除したいプロジェクトのブロックを探す：

```html
<!-- 📌 プロジェクト 2：データベース連携Webアプリ -->
...
...
<!-- 📌 プロジェクト 2 ここまで -->
```

### ステップ2: 選択して削除

上記の**2つのコメントを含めて、その間のすべて**を選択して削除

**⚠️ 注意：**
- `<div>` の開始タグと閉じタグ `</div>` がセットになっているか確認
- 削除後、ファイルを保存して、ブラウザで確認

## 💡 よくある質問

### Q1: HTMLファイルはどのエディタで開けばいいですか？

**A:** 以下のエディタがおすすめです：

**Windows:**
- Visual Studio Code（無料、おすすめ）
- Notepad++（無料）
- メモ帳（標準搭載）

**Mac:**
- Visual Studio Code（無料、おすすめ）
- Sublime Text
- テキストエディット（標準搭載）

**Visual Studio Codeのダウンロード：**
https://code.visualstudio.com/

### Q2: コメントは表示されますか？

**A:** いいえ、HTMLのコメント（`<!-- ... -->`）はブラウザで表示されません。
コメントは**編集するときの目印**なので、そのまま残しておいてOKです。

### Q3: 編集したら表示がおかしくなりました

**A:** 以下を確認してください：

1. **タグの閉じ忘れ**
   - `<div>` を開いたら `</div>` で閉じる
   - `<p>` を開いたら `</p>` で閉じる

2. **ダブルクォート（"）の対応**
   ```html
   <!-- OK -->
   <span class="tag-chip">技術名</span>
   
   <!-- NG（閉じてない） -->
   <span class="tag-chip>技術名</span>
   ```

3. **元に戻す方法**
   - テキストエディタの「元に戻す」（Ctrl+Z / Cmd+Z）を使う
   - GitHubにバックアップがあれば、再ダウンロード

### Q4: プロジェクトは何個まで追加できますか？

**A:** 制限はありません！
ただし、3〜5個程度が見やすくておすすめです。

### Q5: プロジェクトの順番を変えたいです

**A:** プロジェクトのブロック全体を切り取って、好きな場所に貼り付けるだけでOKです：

1. プロジェクトブロック全体を選択（`<!-- 📌 プロジェクト X -->` から `<!-- 📌 プロジェクト X ここまで -->` まで）
2. 切り取り（Ctrl+X / Cmd+X）
3. 移動したい場所にカーソルを置く
4. 貼り付け（Ctrl+V / Cmd+V）

### Q6: 絵文字が表示されません

**A:** ファイルの保存形式を確認してください：

- **文字コード**: UTF-8
- **改行コード**: LF または CRLF

Visual Studio Codeなら、右下に表示されています。

### Q7: GitHubに反映されません

**A:** 以下を確認：

1. ファイルを保存しましたか？
2. GitHubにアップロードしましたか？
3. GitHub Pagesの設定は有効ですか？
4. 数分待ってから更新してみてください

## 🎓 編集のコツ

### 1. まずコピーして試す

- いきなり編集せず、プロジェクトブロックをコピーして試す
- 失敗しても元に戻せる

### 2. こまめに保存

- 編集したら保存（Ctrl+S / Cmd+S）
- ブラウザで確認
- また編集

### 3. バックアップを取る

- 編集前にファイルをコピーしておく
- `index_backup.html` などの名前で保存

### 4. ブラウザの開発者ツールを使う

- ブラウザで右クリック → 「検証」
- エラーがあれば表示される
